# Compiler-precise indexing with syntactic fallback

Call/reference edges come primarily from compiler-grade SCIP indexers (which means the Index Server resolves dependencies and runs builds in sandboxes — accepted cost), because org-wide impact analysis ("what breaks if I change X") is a core promise and name-based matching can't deliver it. Code that SCIP can't cover (unsupported language, broken build) falls back to tree-sitter syntactic extraction, so the graph is never empty for a repo. Every call/reference edge carries provenance (precise | syntactic — the full edge enum adds extracted | inferred, RFC 0001 §5) and a confidence tag, so consumers always know what was proven vs guessed. Rejected: tree-sitter-only (graphify's approach) — maximum breadth but can't be trusted for impact analysis.
